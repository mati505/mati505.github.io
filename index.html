The 3 Basic Web Languages

Webpages need to do a lot of stuff at once, and it can get confusing (very) when you need to manage everything in one code document.

That's why programmers made 3 different main Web languages, each managing a separate part of the webpage. Every webpage builds upon these.

In this section, I will explain a little bit about each one of them, and what order to learn them in:

HTML: Web Backbone

HTML is short for Hyper Text Markup Language, a fancy description for "what the webpage actually contains". It doesn't have any fancy styling, but all of the webpages content is made in HTML

CSS: The Look Lounge

You can say anything you want about HTML, but it definetly isn't pretty. Without CSS, our creations would still be bland, greyish, and not interesting paragraphs.

Javascript: The Action Arena

Who cares if you have nice styles and content? Without Javascript, everything would be so boooring. Could you even survive a day without annoying cat popups?
